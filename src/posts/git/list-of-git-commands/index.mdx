## Create a new Repository on the command line

```shell
git init
git add .
git commit -m "first commit"
git remote add origin https://github.com/mhmtlgly/nameoftherepository.git
git push -u origin master

////

git remote -v

git push
git push --force
git push origin master
git push -u origin master // --set-upstream => always push to master

fatal: remote origin already exists
git remote set-url origin https://github.com/mhmtlgly/testing
```

## Stash

```shell
git stash
git stash -u // includes untracked files
git stash apply
git stash list // display list of all stashes

git stash apply stashNumber // apply stash with stash_number

git stash push -m „stash_title_to_identify_it_easier“
git stash push -u -m „stash_title_to_identify_it_easier“ //include untracked files

git stash drop stashNumber // delete stash
git stash pop stashNumber // apply stash and delete stash afterwards
git stash clear // clears stash list => delete all
```

## Tracking Remote Repo

```shell
git branch -u origin/master

# It depends on what state your local repository is in relative to upstream.
# If there are conflicts, then you are better off stashing your work before pulling your branch in. That can be accomplished thus:
git add . && git stash save
git pull
git stash pop

# rebasing. If you pull remote changes with the flag --rebase , then your local changes are reapplied on top of the remote changes.
# git rebase master is the proper way to do this. Merging would mean a commit would be created for the merge, while rebasing would not.
git pull origin master –rebase
```
